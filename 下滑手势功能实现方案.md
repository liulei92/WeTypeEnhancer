# 按键下滑手势功能实现方案

## 总体架构

### 核心思想
由于 WeType 使用的是 self-draw 自绘键盘（Canvas 绘制，非标准 View 按键），实现方案如下：

1. **标签绘制**：Hook 键盘容器的 `dispatchDraw(Canvas)` 方法（`hookAfter`），直接在 Canvas 上绘制标签文字，不创建额外的 View 覆盖层。更可靠、无生命周期管理负担
2. **手势检测**：Hook 键盘容器的 `dispatchTouchEvent`（`hookBefore`），下滑超阈值时替换事件为 `ACTION_CANCEL` 并执行自定义操作。回调内用 `view !== trackedKeyboardView` 守卫避免全局污染
3. **动作执行**：通过 `InputConnection` API 执行文本编辑操作（复制/粘贴/全选等），调用前检查 null

### 获取键盘 View 的路径
```
InputMethodService.onStartInputView
  → ims.currentInputView  (直接属性访问，无需反射)
```

### 对象 `WeTypeSwipeGestureHooks`

#### 核心 Hook 入口
```kotlin
fun hookSwipeGesture()
```
1. **在 `MainHook.installWeTypeHooks()` 时注册 Hook**，Hook 回调内部检查开关
2. Hook `InputMethodService.onStartInputView(EditorInfo, Boolean)`:
   - **入口检查**：`if (!WeTypeSettings.isSwipeGestureEnabledXposed()) return`
   - 获取 InputMethodService 实例
   - 通过反射获取 `getInputView()` 得到键盘容器 View
   - **Hook `dispatchDraw(Canvas)` — 标签绘制入口**：`hookAfter`，在 Canvas 上绘制按键标签
   - **Hook `dispatchTouchEvent(MotionEvent)` — 手势检测入口**：`hookBefore`，监听触摸拦截下滑
3. Hook `InputMethodService.onFinishInputView(Boolean)`:
   - 清理状态（触摸状态、键盘 View 引用）
4. **开关关闭时**：
   - 不执行任何手势相关代码，零性能开销

#### 标签绘制 (dispatchDraw hookAfter)
```kotlin
view.javaClass.getMethod("dispatchDraw", Canvas::class.java).hookAfter { param ->
    val canvas = param.args[0] as? Canvas ?: return@hookAfter
    val v = param.thisObject as? View ?: return@hookAfter
    // 入口守卫
    if (!WeTypeSettings.isSwipeGestureEnabledXposed()) return@hookAfter
    
    val layout = detectKeyboardLayout(v)
    val configs = parseGestureConfig()
    val paint = createLabelPaint(v)  // 颜色、字号从设置读取
    
    for ((keyLabel, action) in configs) {
        val bounds = calculateKeyBounds(v, layout, keyLabel) ?: continue
        // 在按键底部绘制半透明背景 + 文字
        drawLabel(canvas, bounds, action.labelShort, paint)
        // 注意：使用 labelShort（双字缩写），而非 labelZh
    }
}
```

**只读操作，无副作用，不修改原 Canvas 状态**。只在开关开启且键盘 View 匹配时绘制，不额外创建 View。

#### 按键位置计算
QWERTY 各行星键数量不等，需逐行计算列数和左缩进：
```kotlin
data class KeyPosition(val row: Int, val col: Int, val label: String)

// QWERTY 每行键数（不含功能键）
val QWERTY_ROW_COLS = intArrayOf(10, 9, 7)  // Q-P行10键, A-L行9键(左缩进20%), Z-M行7键(左缩进30%)

val QWERTY_KEYS = listOf(
    "Q","W","E","R","T","Y","U","I","O","P",  // 行0, 10列
    "A","S","D","F","G","H","J","K","L",     // 行1, 9列, 左缩进
    "Z","X","C","V","B","N","M"               // 行2, 7列, 更深缩进
)

// T9 按键列表（仅可配置的 2-9）
val T9_KEYS_CONFIGURABLE = listOf("2","3","4","5","6","7","8","9")
```

**位置估算规则**（标签只需大致位置，偏差 1-2px 无影响）：
- 容器宽 / 该行星数 = 每键宽度，乘以列索引 + 缩进偏移 = x 坐标
- 容高 / 4行 = 每行高度，乘以行索引 = y 坐标
- 标签绘制在按键底部区域（y + 行高 * 0.7）

布局判定：检测键盘容器宽高比，宽 < 高 * 1.2 → T9 (3列)，否则 QWERTY

#### 手势检测 (dispatchTouchEvent hookBefore)
```kotlin
// 获取键盘 View 的具体类上的 dispatchTouchEvent 方法
// 如果具体类没重写，得到的是 View.dispatchTouchEvent，需要实例守卫
val method = view.javaClass.getMethod("dispatchTouchEvent", MotionEvent::class.java)
method.hookBefore { param ->
    // 【安全守卫】确保只处理目标键盘 View
    if (param.thisObject !== targetKeyboardView) return@hookBefore
    if (!WeTypeSettings.isSwipeGestureEnabledXposed()) return@hookBefore
    
    val event = param.args[0] as? MotionEvent ?: return@hookBefore
    when (event.action and MotionEvent.ACTION_MASK) {
        ACTION_DOWN -> {
            recordPosition(view, event.x, event.y)
        }
        ACTION_MOVE -> {
            val dy = event.y - lastDownY
            if (dy > thresholdPx && isTracking) {
                // ① 将当前事件替换为 ACTION_CANCEL（键盘重置状态）
                val cancel = MotionEvent.obtain(event)
                cancel.action = MotionEvent.ACTION_CANCEL
                param.args[0] = cancel
                // ② 设置截获标记，后续事件全部消耗
                setIntercepted(view, true)
                // ③ 执行动作
                val action = resolveAction(event.x, event.y)
                executeAction(action)
                // ④ 视觉反馈：触发后标记需要闪烁，dispatchDraw 时放大文字绘制
                triggerFlashFeedback()
            }
        }
        ACTION_UP, ACTION_CANCEL -> {
            if (isIntercepted) param.result = true
            cleanup(view)
        }
    }
    // 截获期间后续 MOVE/UP 直接消耗（修正：action 改为 event.action）
    if (isIntercepted && (event.action and MotionEvent.ACTION_MASK) != MotionEvent.ACTION_DOWN) param.result = true
}
```

**关键安全措施**：
- `thisObject !== targetKeyboardView` 守卫 → 防止 Hook 全局 View.dispatchTouchEvent 污染 IME 进程所有 View
- 下滑阈值从 15dp 提升到 20dp → 减少与 WeType 自身下滑手势的冲突

#### 视觉反馈（触发闪烁）
手势触发后，被触发按键的标签文字短暂闪烁一次：
```kotlin
// flashTriggerKey: 当前触发的按键（如 "Q"）
// flashStartTime: 闪烁开始时间戳
// flashDuration: 100ms（短暂闪烁）

// 在 dispatchDraw hookAfter 中：
val elapsed = System.currentTimeMillis() - flashStartTime
if (flashTriggerKey != null && elapsed < flashDuration) {
    // 前50ms 隐藏标签，后50ms 显示标签
    val isVisible = elapsed >= (flashDuration / 2)
    if (!isVisible) skipDrawingForKey(flashTriggerKey)
} else {
    flashTriggerKey = null  // 闪烁结束
}
```
实现极其简单——只在闪烁前半段跳过该按键的标签绘制即可。闪烁状态在 `onFinishInputView` 时清理。

#### 动作执行
通过 InputConnection，**调用前检查 null**：
```kotlin
val ims = currentInputMethodService ?: return  // from onStartInputView
val ic = runCatching {
    ims.invokeMethodAs<InputConnection>("getCurrentInputConnection")
}.getOrNull() ?: return  // 【null 安全检查】

when (action) {
    LINE_START -> {
        ic.sendKeyEvent(KeyEvent(KeyEvent.ACTION_DOWN, KeyEvent.KEYCODE_MOVE_HOME))
        ic.sendKeyEvent(KeyEvent(KeyEvent.ACTION_UP, KeyEvent.KEYCODE_MOVE_HOME))
    }
    LINE_END -> {
        ic.sendKeyEvent(KeyEvent(KeyEvent.ACTION_DOWN, KeyEvent.KEYCODE_MOVE_END))
        ic.sendKeyEvent(KeyEvent(KeyEvent.ACTION_UP, KeyEvent.KEYCODE_MOVE_END))
    }
    SELECT_ALL -> ic.performContextMenuAction(android.R.id.selectAll)
    COPY       -> ic.performContextMenuAction(android.R.id.copy)
    CUT        -> ic.performContextMenuAction(android.R.id.cut)
    PASTE      -> ic.performContextMenuAction(android.R.id.paste)
    CLEAR      -> {
        // 全选后删除 = 清空内容
        ic.performContextMenuAction(android.R.id.selectAll)
        ic.performContextMenuAction(android.R.id.cut)
    }
    CLIPBOARD  -> {
        // 方案A：尝试系统剪贴板 Intent（Android 10+）
        runCatching {
            val intent = Intent().apply {
                action = "android.intent.action.PASTE"  // 部分 ROM 支持
            }
            ims.startActivity(intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK))
        }
        // 方案B：通过反射启动 WeType 内部剪贴板 Activity
        // 需在实现时通过 WeType 类搜索确认具体类名
    }
    HIDE_KEYBOARD -> {
        // 通过 InputMethodManager 隐藏键盘
        // 注意：必须用 ims.getWindow() 获取 Window 对象
        val imm = ims.getSystemService(InputMethodManager::class.java)
        imm.hideSoftInputFromWindow(ims.getWindow().getDecorView().getWindowToken(), 0)
    }
    SWITCH_IME -> {
        val imm = ims.getSystemService(InputMethodManager::class.java)
        // 注意：必须用 ims.getWindow() + getAttributes().token
        val token = ims.getWindow().getAttributes().token
        imm.switchToNextInputMethod(token, false)
    }
    SETTINGS   -> {
        // 打开 IME 设置页面（使用 WeType 自身 Activity 或系统 IME 设置）
        val intent = Intent(Settings.ACTION_INPUT_METHOD_SETTINGS)
        context.startActivity(intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK))
    }
}
```

**InputConnection 在任何时刻都可能为 null**（Window 销毁/重建间隙），每次执行动作必须检查。

#### 导入（全量覆盖）
1. 读取剪贴板文本
2. 解析 JSON → 失败则 Toast `配置格式无效`，中止
3. **校验每个字段**：
   - `enabled` 必须是 boolean
   - `threshold` 必须是 Int 且在 8~50 范围

4. 任何一项校验失败 → Toast `配置内容无效`，中止
5. 全部通过 → 填充到 UI 状态，Toast `已从剪贴板恢复`
6. 不自动保存，需用户点击保存按钮

## 七、边界情况处理

1. **总开关关闭时**（默认）：Hook 回调内部直接 return，不绘制标签、不处理触摸，零性能开销
2. **dispatchTouchEvent 实例守卫**：`param.thisObject !== targetView` 时 return，防止全局 View.dispatchTouchEvent 污染
3. **InputConnection 为 null**：每次执行动作前检查，null 时静默跳过
4. **无配置的按键**：只遍历已配置的按键绘制标签，不遍历全键盘
5. **硬件键盘连接**：不复用覆盖层（不需要了），但在 dispatchDraw 中检测并跳过绘制
6. **手写/语音输入模式**：检测 keyboardView 的子 View 或状态，跳过标签绘制
7. **dispatchDraw 不是 ViewGroup 专有**：通过 try-catch 捕获 `NoSuchMethodException`，fallback 到 hook `onDraw(Canvas)`
8. **JSON 缓存线程安全**：`getSwipeGestureMapXposed()` 的结果缓存使用 `ConcurrentHashMap` 替代 `WeakHashMap`，避免并发问题
9. **onFinishInputView 清理**：清除触摸状态缓存、JSON 缓存、键盘 View 引用、闪缩状态，防止内存泄漏
