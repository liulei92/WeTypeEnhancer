# 按键下滑手势功能实现方案

## 总体架构

### 核心思想
由于 WeType 使用的是 self-draw 自绘键盘（Canvas 绘制，非标准 View 按键），实现方案如下：

1. **标签绘制**：Hook 键盘容器的 `dispatchDraw(Canvas)` 方法（`hookAfter`），直接在 Canvas 上绘制标签文字，不创建额外的 View 覆盖层。更可靠、无生命周期管理负担
2. **手势检测**：Hook 键盘容器的 `dispatchTouchEvent`（`hookBefore`），下滑超阈值时替换事件为 `ACTION_CANCEL` 并执行自定义操作。回调内用 `view !== trackedKeyboardView` 守卫避免全局污染
3. **动作执行**：通过 `InputConnection` API 执行文本编辑操作（复制/粘贴/全选等），调用前检查 null

### 获取键盘 View 的路径
```
InputMethodService.onStartInputView
  → ims.invokeMethodAs<View>("getCurrentInputView")  (反射调用)
```

> **注意**：`getCurrentInputView()` 不是公共 API，在 compileSdk 37 中不可直接访问。
> 必须使用 `invokeMethodAs` 反射调用。

### 核心 Hook 入口：`hookSwipeGesture()`

```kotlin
fun hookSwipeGesture() {
    hookIMSLifecycle()          // IMS 生命周期追踪 + 动态注册 dispatch hooks
}
```

在 `MainHook.installWeTypeHooks()` 时注册。所有 Hook 回调内部检查开关 `isSwipeEnabledXposed()`，关闭时零性能开销。

**架构变更说明（v2 简化）**：
- 不再有静态的 `hookViewDispatchDraw()` 和 `hookViewDispatchTouchEvent()`
- dispatch hooks 改为在 `onStartInputView` 获取到键盘 View 实例后动态注册
- 原因：ViewGroup 重写了 View 的 dispatchDraw/dispatchTouchEvent，Xposed 只 hook 精确的 ArtMethod。
  静态 hook View 的方法不会在 ViewGroup 实例上触发。
- 解决方案：使用 `findMethodInHierarchy` 在键盘 View 的实际类中找到最子类的重写版本再 hook

### IMS 生命周期 Hook (`hookIMSLifecycle`)

```kotlin
// onStartInputView → trackedKeyboardView = ims.invokeMethodAs<View>("getCurrentInputView")
//                  → initDynamicDispatchHooks(keyboardView)  ← 动态注册 dispatch hooks
// onFinishInputView → 清理状态
// onDestroy → 完整清理
```

**关键实现**：
- 使用 `loadClassOrNull("android.inputmethodservice.InputMethodService")` + `findMethod` 反射获取方法
- 参数类型比较用 `java.lang.Boolean.TYPE`（即 `boolean.class`）而非 `Boolean::class.javaPrimitiveType`
- `trackedKeyboardView = ims.invokeMethodAs<View>("getCurrentInputView")`（反射调用）
- `initDynamicDispatchHooks(keyboardView)` 在获取到 View 实例后调用，通过 `viewInstance.javaClass.findMethodInHierarchy` 查找最子类的 dispatchDraw/dispatchTouchEvent 方法并 hook
- 使用 `dispatchHooksInitialized` 标志（`@Volatile`）确保只 hook 一次

### 标签绘制 (`initDynamicDispatchHooks` → dispatchDraw hookAfter)

```kotlin
// dispatchDraw hookAfter（动态注册，在 initDynamicDispatchHooks 中执行）
clazz.findMethodInHierarchy {
    name == "dispatchDraw" && parameterTypes.sameAs(Canvas::class.java)
}.hookAfter { param ->
    if (!WeTypeSettings.isSwipeEnabledXposed()) return@hookAfter
    val view = param.thisObject as? View ?: return@hookAfter
    if (view !== trackedKeyboardView) return@hookAfter  // 引用比较守卫
    val canvas = param.args.getOrNull(0) as? Canvas ?: return@hookAfter
    
    // 闪烁处理：前50ms跳过绘制
    if (!shouldDrawLabel()) return@hookAfter
    
    // 绘制标签
    drawKeyLabels(view, canvas)
}
```

> **注**：不再使用静态 `hookViewDispatchDraw()`，而是通过 `initDynamicDispatchHooks(viewInstance)` 在运行时
> 获取键盘 View 的实际 `javaClass`，用 `findMethodInHierarchy` 找到最子类的 dispatchDraw 再 hook。

### 键位映射与位置计算

```kotlin
private data class KeyPos(val row: Int, val col: Int)

// QWERTY 字母键位置映射
val QWERTY_KEYS = mapOf(
    "Q" to KeyPos(0,0), ..., "P" to KeyPos(0,9),
    "A" to KeyPos(1,0), ..., "L" to KeyPos(1,8),
    "Z" to KeyPos(2,0), ..., "M" to KeyPos(2,6)
)

// QWERTY 行配置
val QWERTY_ROW_COLS = intArrayOf(10, 9, 7)
val QWERTY_ROW_INDENTS = floatArrayOf(0f, 0.05f, 0.15f)  // 缩进比例

// T9 可配按键（第1~3行，3列）
val T9_KEYS = mapOf(
    "2" to KeyPos(1,0), "3" to KeyPos(1,1),
    "4" to KeyPos(2,0), ..., "9" to KeyPos(3,2)
)

// 布局判定（宽高比 < 1.2 → T9）
private fun isT9Keyboard(width: Int, height: Int): Boolean =
    width > 0 && height > 0 && width < height * 1.2f

// 按键中心坐标计算
private fun keyCenter(key: String, pos: KeyPos, viewW: Int, viewH: Int, isT9: Boolean): Pair<Float, Float>
```

**坐标计算规则**：
- **QWERTY**：假设 4 行（含数字行），字母从第 1 行开始。每行按配置的列数 + 缩进比例计算，
  垂直居中 (`cy = (pos.row + 1) * cellH + cellH / 2f`)
- **T9**：5 行 × 3 列，无缩进
- 使用 `fontMetrics` 实现文字精确垂直居中 (`cy - (ascent + descent) / 2f`)

### 手势检测 (`initDynamicDispatchHooks` → dispatchTouchEvent hookBefore)

```kotlin
// dispatchTouchEvent hookBefore（动态注册，在 initDynamicDispatchHooks 中执行）
clazz.findMethodInHierarchy {
    name == "dispatchTouchEvent" && parameterTypes.sameAs(MotionEvent::class.java)
}.hookBefore { param ->
    // 守卫：开关 + View 引用比较
    if (!WeTypeSettings.isSwipeEnabledXposed()) return@hookBefore
    val view = param.thisObject as? View ?: return@hookBefore
    if (view !== trackedKeyboardView) return@hookBefore

    val event = param.args[0] as? MotionEvent ?: return@hookBefore
    when (event.action and MotionEvent.ACTION_MASK) {
        MotionEvent.ACTION_DOWN -> {
            resetTouch()
            touchDownY = event.y
            touchDownKey = estimateKeyAt(view, event.x, event.y)
        }
        MotionEvent.ACTION_MOVE -> {
            if (intercepted) return@hookBefore
            val dy = event.y - touchDownY
            val thresholdPx = TypedValue.applyDimension(...)
            if (dy >= thresholdPx) {
                val key = touchDownKey ?: return@hookBefore       // 检查触摸在键上
                val actionConfig = keyMap[key] ?: return@hookBefore // 检查键有配置
                intercepted = true
                // 替换事件为 ACTION_CANCEL
                val cancelEvent = MotionEvent.obtain(
                    event.downTime, event.eventTime,
                    MotionEvent.ACTION_CANCEL, event.x, event.y, event.metaState
                )
                param.args[0] = cancelEvent
                executeAction(actionConfig, service, context)
                flashStartMs = System.currentTimeMillis()
                view.invalidate()
            }
        }
        MotionEvent.ACTION_UP, MotionEvent.ACTION_CANCEL -> {
            resetTouch()
        }
    }
}
```

> **注**：不再使用静态 `hookViewDispatchTouchEvent()`，改为在 `initDynamicDispatchHooks` 中动态注册。

**关键安全措施**：
- `view !== trackedKeyboardView` 守卫 → 防止 Hook 影响 IME 进程所有 View
- `touchDownKey ?: return@hookBefore` + `keyMap[key] ?: return@hookBefore` → 仅在**已配置动作的按键**上拦截
- `MotionEvent.obtain(downTime, eventTime, ACTION_CANCEL, x, y, metaState)` 创建取消事件
- 阈值从默认 20dp 开始，用户可在 8~50dp 范围调节

#### 按键识别 `estimateKeyAt`
根据触摸坐标遍历键位映射，判断触摸点是否在按键范围内：
```kotlin
for ((keyLabel, pos) in keyMap) {
    val (cx, cy) = keyCenter(keyLabel, pos, ...)
    // 计算 cellW / cellH
    if (x in (cx - halfW)..(cx + halfW) && y in (cy - halfH)..(cy + halfH)) {
        return keyLabel
    }
}
return null
```

### 视觉反馈（触发闪烁）

```kotlin
private const val FLASH_DURATION_MS = 100L  // 总闪烁 100ms
private const val FLASH_HIDE_MS = 50L       // 前 50ms 隐藏

private fun shouldDrawLabel(): Boolean {
    if (flashStartMs == 0L) return true
    val elapsed = System.currentTimeMillis() - flashStartMs
    if (elapsed >= FLASH_DURATION_MS) { flashStartMs = 0L; return true }
    return elapsed >= FLASH_HIDE_MS  // 前50ms隐藏，后50ms显示
}
```

在 `dispatchDraw` 中调用：闪烁期间前 50ms 跳过所有标签绘制（`return@hookAfter`），后 50ms 恢复。
**不跟踪具体哪个按键触发**，整键盘闪烁。

### 动作执行

通过 `service.currentInputConnection`（**每次调用前检查 null**）：

```kotlin
private fun executeAction(action: SwipeAction, service: InputMethodService, context: Context) {
    try {
        when (action) {
            SELECT_ALL  → service.currentInputConnection?.performContextMenuAction(android.R.id.selectAll)
            COPY        → service.currentInputConnection?.performContextMenuAction(android.R.id.copy)
            CUT         → service.currentInputConnection?.performContextMenuAction(android.R.id.cut)
            PASTE       → service.currentInputConnection?.performContextMenuAction(android.R.id.paste)
            CLEAR       → { selectAll + cut }
            LINE_START  → sendKeyEvent(service, KEYCODE_MOVE_HOME)
            LINE_END    → sendKeyEvent(service, KEYCODE_MOVE_END)
            CLIPBOARD   → openClipboard(context)     // Intent("android.intent.action.CLIPBOARD")
            HIDE_KEYBOARD → hideKeyboard(service)    // invokeMethodAs("getWindow") → getWindow()
            SWITCH_IME  → switchIME(service)          // 同上
            SETTINGS    → openIMESettings(context)    // Settings.ACTION_INPUT_METHOD_SETTINGS
        }
    } catch (e: Exception) { Log.e(...) }
}
```

**辅助函数**：
- `sendKeyEvent(service, keyCode)` → 通过 `service.currentInputConnection.sendKeyEvent(down/up)`
- `hideKeyboard` → `invokeMethodAs<Any>("getWindow")` → `getWindow()` → `decorView.windowToken` → `InputMethodManager.hideSoftInputFromWindow()`
- `switchIME` → 同上获取 token → `InputMethodManager.switchToNextInputMethod()`
- `openClipboard` → `Intent("android.intent.action.CLIPBOARD")` + 设置 `package = "com.tencent.wetype"`
- `openIMESettings` → `Intent(Settings.ACTION_INPUT_METHOD_SETTINGS)`

**InputConnection 在任何时刻都可能为 null**（Window 销毁/重建间隙），每次执行动作前检查 null。

## 七、边界情况处理

1. **总开关关闭时**（默认）：Hook 回调内部直接 return，不绘制标签、不处理触摸，零性能开销
2. **View 引用守卫**：`view !== trackedKeyboardView` 时 return，防止全局 View.dispatchTouchEvent/dispatchDraw 污染
3. **InputConnection 为 null**：通过 `service.currentInputConnection?` 安全调用，null 时静默跳过
4. **仅绘配置按键**：`parseKeyMap()` 返回 `Map<String, SwipeAction>`，只遍历已配置的按键，不遍历全键盘
5. **非配置区不下滑拦截**：`touchDownKey ?: return@hookBefore` + `keyMap[key] ?: return@hookBefore` 双重检查，仅在按在已配键上才触发
6. **MotionEvent.obtain 不泄露**：创建的 cancelEvent 直接替换 `param.args[0]`，Android 事件池管理回收
7. **dispatchDraw 不影响非键盘 View**：`view !== trackedKeyboardView` 引用守卫确保不受影响
8. **无 Drawable/View 层泄漏**：纯 Canvas.drawText 绘制，不创建额外的 View 或 Drawable 对象
9. **onFinishInputView 清理**：清除触摸状态、键盘 View 引用、闪烁时间戳，防止泄漏
10. **onDestroy 完整清理**：同上，确保 IMS 销毁后无残留
