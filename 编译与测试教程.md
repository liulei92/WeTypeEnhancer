# WeTypeEnhancer — 编译与测试教程

> 项目仓库：[https://github.com/liulei92/WeTypeEnhancer](https://github.com/liulei92/WeTypeEnhancer)

## 一、环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| **JDK** | **21** | 必须为 JDK 21，低版本或高版本均不兼容 |
| **Android Studio** | 最新版 (2024.x+) | 推荐安装最新稳定版 |
| **Android SDK** | **API 37 (Android 16)** | 编译目标版本，需要在 SDK Manager 中安装 |
| **Gradle** | 9.4.1 (由 Gradle Wrapper 自动管理) | 无需手动安装，项目已包含 wrapper |

### 1.2 环境检查

```powershell
# 检查 JDK 版本
java -version
# 应输出类似：openjdk version "21" ...

# 检查 Android Studio 中已安装的 SDK 平台
# 打开 Android Studio → Settings → Languages & Frameworks → Android SDK → SDK Platforms
# 确保 "Android 16.0 (API 37)" 已勾选安装
```

### 1.3 项目导入

1. **克隆/打开项目**
   - 启动 Android Studio
   - `File` → `New` → `Project from Version Control...` → 输入仓库地址
   - 或 `File` → `Open...` → 选择本地项目目录
   - 等待 Gradle 同步完成（首次同步需要下载依赖，时间较长）

2. **SDK 配置**（如果没有自动检测到 SDK）
   - `File` → `Project Structure` → `SDK Location`
   - 确认 `Android SDK location` 指向你的 SDK 安装路径
   - 例如：`C:\Users\<用户名>\AppData\Local\Android\Sdk`

## 二、使用 GitHub Actions 自动编译（推荐，无需本地环境）

> 这是**最推荐的方式**，不需要在本地安装 JDK、Android SDK 或 Android Studio，
> 所有编译都在 GitHub 服务器上完成，你只需下载编译好的 APK 即可。

### 2.1 如何触发编译

将项目推送到 GitHub 后，以下情况会自动触发编译：

| 触发方式 | 操作 | 说明 |
|---------|------|------|
| **推送代码** | `git push` 到 `main` 或 `master` 分支 | 每次提交自动编译 Debug + Release APK |
| **创建 PR** | 提交 Pull Request 到 `main` 分支 | 自动编译验证代码是否正确 |
| **手动触发** | GitHub 页面 → Actions → Build APK → `Run workflow` | 无需提交代码，随时手动编译 |
| **推送标签** | `git tag v1.0.0 && git push --tags` | 编译并自动创建 Release 发布页 |

### 2.2 下载 APK（从 Actions 产物）

1. 打开项目的 GitHub 页面
2. 点击 **Actions** 标签
3. 点击最新的成功工作流运行
4. 在 **Artifacts** 区域下载：
   - `WeType-UI-Enhanced-Debug` → 调试版 APK（日常使用）
   - `WeType-UI-Enhanced-Release` → 发布版 APK（含混淆优化）

### 2.3 下载 APK（从 Releases）

当维护者推送 `v*` 格式的标签时（如 `v1.0.0`），会自动创建 GitHub Release：

1. 打开项目 GitHub 页面
2. 点击右侧的 **Releases** 区域
3. 找到对应的版本，下载 `.apk` 附件即可

### 2.4 首次使用前：将项目推送到 GitHub

如果你是项目维护者，首次使用需要将代码推送到 GitHub：

1. 在 GitHub 上创建一个新仓库（**不要勾选** README、.gitignore 等初始化选项）
2. 在本地项目根目录执行：
   ```powershell
   git remote add origin https://github.com/<你的用户名>/<仓库名>.git
   git push -u origin main
   ```
3. 推送后，GitHub 会自动识别 `.github/workflows/build.yml`，工作流即生效
4. **以后每次推送代码**，APK 都会自动编译，你只需从 Actions 页面下载即可

### 2.5 手动触发编译

也可以在 GitHub 网页上点击按钮手动编译，无需提交代码：

```
GitHub 仓库 → Actions → Build APK → [Run workflow] → 选择分支 → [Run]
```

等待约 2-3 分钟编译完成后，Artifacts 区就会出现 APK 下载链接。

### 2.6 注意事项

- **Node.js 兼容性警告**：Actions 运行时会提示 `Node.js 20 is deprecated` 等信息，这是 GitHub 升级运行时的正常警告，**不影响编译结果**
- **Gradle 弃用警告**：构建日志中会出现多个 `WARNING`（如 `android.builtInKotlin=false` 等），均为 AGP 9.x 的兼容性提示，**不影响 APK 的正常生成**

# 编译 Release APK（含混淆和资源压缩）
.\gradlew assembleRelease

# 完整构建
.\gradlew build

# 清理
.\gradlew clean
```

```

## 四、Xposed 模块安装与激活

### 4.1 运行环境要求

| 条件 | 要求 |
|------|------|
| **设备** | 已 Root 的小米 HyperOS / MIUI 设备（或 Android 14+ 模拟器） |
| **Xposed 框架** | **LSPosed**（API >= 93） |
| **Android 版本** | Android 14 (API 31) 及以上 |
| **目标应用** | 微信输入法已安装 |
| **模块作用域** | 需在 LSPosed 中勾选输入法进程 |

### 4.2 安装步骤

#### 从 GitHub Actions 下载 APK 后安装
```powershell
# 将 APK 推送到手机
adb install -r WeType_UI_Enhanced-1.26.0_debug.apk
```

#### 从本地编译安装
```powershell
adb install -r app\build\outputs\apk\debug\WeType_UI_Enhanced-1.26.0_debug.apk
```

#### 在 LSPosed 中激活
1. 打开 LSPosed Manager
2. 进入 `模块` 页面
3. 找到 `WeType UI Enhanced` 模块，**勾选启用**
4. **作用域** → 勾选 `微信键盘`（com.tencent.wetype）
5. 强制停止微信键盘（或重启设备）

#### 验证激活
- 打开输入法设置页面，如果能正常显示模块的配置界面，即表示模块加载成功

### 4.3 调试安装（带日志）

```powershell
# 实时过滤 WeType 相关日志
adb logcat -c
adb logcat | findstr "WeTypeHook\|wetypehook"
```

## 五、测试方法

### 5.1 模块自身功能测试

该模块提供两部分功能：

**① 模块设置页面（宿主 App）**
- 安装并激活后，桌面会出现应用图标 `WeType UI Enhanced`
- 打开可查看和修改模块的各项配置：外观颜色、下滑手势等

**② 输入法运行时 Hook 功能**
- 在任意输入框中唤起微信键盘
- 可观察到的效果：
  - **外观定制**：键盘颜色、高光效果、按键圆角等已应用修改
  - **下滑手势**（如已启用）：在配置了手势的按键上向下滑动，触发指定操作

### 5.2 功能验证清单

| 功能模块 | 验证方法 | 预期结果 |
|---------|---------|---------|
| **模块加载** | 打开模块设置页面 | 正常显示配置界面 |
| **外观修改** | 在输入框中唤出键盘 | 键盘颜色/主题已改变 |
| **下滑手势** | 在设置中启用，在按键上向下滑动 | 执行对应操作（复制/粘贴等） |

### 5.3 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|-----|---------|---------|
| 模块设置页空白 | 模块未正常加载 | 检查 LSPosed 中模块是否启用，作用域是否正确 |
| 键盘外观无变化 | Hook 未生效 | 强制停止微信键盘后重新唤起 |
| 下滑手势无反应 | 开关未开启/阈值过大 | 在设置中确认已启用，尝试减小阈值 |
| 编译失败（本地） | JDK 版本不对 | 确认使用 JDK 21 |
| 编译失败（本地） | SDK 缺失 | 在 SDK Manager 中安装 API 37 |
| 编译失败（Actions） | 查看 Actions 日志 | 贴出错误日志排查 |

### 5.4 日志抓取

模块在运行时可以通过 logcat 输出调试信息：

```powershell
# 查看所有模块相关日志（标签为 WeTypeHook）
adb logcat -s "WeTypeHook"

# 过滤所有模块相关日志
adb logcat -v time | findstr "wetypehook\|WeTypeHook"
```

模块内部日志均通过 `com.xposed.wetypehook.xposed.Log` 输出，标签为 `WeTypeHook`。
